# Сценарий видео — Advanced-курс, День 1 (Rules)

Видео без звука (та же схема, что для Дня 35 aiexperiment) — весь смысл через
титры/надписи, накладываемые при монтаже (ffmpeg `drawtext`) поверх записи
экрана. Общая длительность — ориентировочно 3-4 минуты.

## 1. Проблема / задача (15-20 сек)

Титр:

> Задание: проанализировать свой проект и создать файл правил для ИИ-ассистента
> (CLAUDE.md) — стек, архитектура, паттерны, антипаттерны. Проверить на боевой
> задаче одним промптом, доработать и сравнить результат до/после.

Показать в кадре структуру проекта CaptainBlood (дерево файлов в IDE) — 3-5 сек,
без чтения, просто фон для контекста «это реальный, а не игрушечный проект».

## 2. Файл правил — CLAUDE.md (40-60 сек)

Прокрутить `CLAUDE.md` по разделам, задерживаясь на каждом заголовке 3-5 сек
(не читать целиком в кадре):

- `## Стек`, `## Архитектура`, `## Структура папок`, `## Naming conventions`
- `## Паттерны проекта` — титр: «Паттерны выведены из реального кода, не выдуманы»
- `## Хороший код — примеры` — задержаться на примере 1 (`RecoveryAction.kt`)
- `## Антипаттерны — запрещено` — задержаться на пункте про SQL/prepareStatement
- `## Шаблон типичного файла`
- `## Как работать над этим проектом с Claude Code` — титр: «Глобальный конфиг,
  локальный конфиг, субагенты, скиллы — описаны отдельно»

## 3. Боевая задача — генерация №1 (40-60 сек)

Титр:

> Один промпт, без уточнений: «Добавь в captainblood новый HTTP-эндпоинт
> GET /stats, который возвращает JSON со статистикой индекса».

Показать сам момент постановки задачи ассистенту (экран с промптом) → результат
(диф в `Main.kt`, добавленный `handleStats`). Титр рядом с диффом:

> Скомпилировалось. Но: без обоснования авторизации, без rate-limit, README не
> обновлён — решения приняты молча, не из правил.

## 4. Доработка правил (20-30 сек)

Показать добавленный раздел `## Чек-лист: новый HTTP-эндпоинт` в `CLAUDE.md`
(diff файла правил, до/после). Титр:

> v1 → v2: добавлен явный чек-лист из трёх вопросов — авторизация, rate-limit,
> синхронизация README. Теперь ассистент обязан ответить на них явно.

## 5. Боевая задача — генерация №2 (40-60 сек)

Тот же промпт, слово в слово, показать в кадре. Результат — diff `Main.kt` +
`README.md`. Титр рядом с KDoc нового эндпоинта:

> Тот же промпт, обновлённые правила: KDoc теперь явно отвечает на все три
> пункта чек-листа. README обновлён без напоминания.

## 6. Сравнение и вывод (20-30 сек)

Показать таблицу сравнения из `docs/AdvancedCourse_Day1_Rules.md` (можно как
скролл markdown-файла или отдельный слайд). Финальный титр:

> Главный вывод: общий стиль кода не гарантирует, что ассистент вспомнит про
> артефакты вне кода и объяснит пограничные решения. Явный чек-лист конкретных
> вопросов повлиял на результат больше, чем правила стиля.

## Заметки на память

- Реальный IP VPS в этой записи не показывать/не упоминать — в коде он уже
  убран в `config.properties` (см. коммит про `VPS_CLIENT_BASE_URL`), но в
  `docs/VPS_Deploy_CaptainBlood.md` он ещё виден в кадре, если скроллить этот
  файл — не открывать его в записи.
- Технический сбой с `isolation: "worktree"` (не создалась отдельная копия
  репозитория при первом прогоне) — не показывать в видео, это внутренняя
  кухня инструмента Claude Code, не часть содержания задания.
