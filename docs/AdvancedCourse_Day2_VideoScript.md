# Сценарий видео — Advanced-курс, День 2 (Profiles)

Видео без звука (та же схема, что для Дня 1 / Дня 35 aiexperiment) — смысл
передаётся титрами, накладываемыми при монтаже (ffmpeg `drawtext`) поверх
записи экрана. Ориентировочно 3.5-5 минут — дольше Дня 1, т.к. вместо
статичного диффа здесь три живых прогона агентов.

## 0. Что попадает в кадр, а что нет (не для озвучки, для себя)

**Обновлено после отката (2026-07-22).** Состояние репозитория откачено до
точки ДО всякого тестирования: `Main.kt` — баг гонки снова в коде
(`waitForOllamaReady()` убран), `README.md`/`CLAUDE.md` — тестовая правка
Docs Sync снята полностью. Значит прогон на записи — **не реконструкция
теста, а настоящий первый прогон**: тестовые запуски 2026-07-21 через
`general-purpose`-сабагентов с профилем в промпте (не нативный
`subagent_type`) были ориентиром/черновой проверкой самого текста профилей,
но не гарантией, что нативный вызов (`@agent-...`) в реальном терминале даст
буква-в-букву тот же результат.

**Запись сделана (2026-07-22, три прогона). Титры ниже — уже по факту
реального вывода**, не по черновой проверке — черновая версия (`waitForOllamaReady()`,
таймаут 60с) разошлась с фактом (`waitForOllamaServer()`, 12×5с ретрай, ~55с,
другой механизм причины — не "прогрев провалился без повтора", а "процесс
целиком падал и передёргивался systemd'ом") и переписана ниже под реальный
результат.

## 1. Задача (15-20 сек)

Титр:

> Задание: два профиля агентного режима — Bug Fix и Research — с системным
> промптом, чек-листом "должен/не должен" и форматом ответа. Плюс третий,
> под свою частую задачу. Проверить на реальных задачах проекта.

## 2. Профили — короткий обзор (30-40 сек)

Прокрутить в IDE три файла `.claude/agents/*.md`, по 3-5 сек на файл, без
чтения вслух — задержаться на структуре (frontmatter → "Ты ДОЛЖЕН" → "Ты НЕ
ДОЛЖЕН" → "Формат ответа"), не на деталях текста. Титр:

> Три профиля: Bug Fix (Read+Edit+Bash), Research (только чтение — без
> Edit/Bash), Docs Sync (документация — код не трогает)

## 3. Bug Fix — живой прогон (40-70 сек)

Промпт — вводится дословно в терминал (одной строкой):

```
@agent-bug-fix После перезагрузки VPS иногда первые один-два вопроса капитану Бладу получают шаблонный ответ вроде "капитан только проснулся, попробуй через полминутки" вместо настоящего — и это может повторяться, приходится спрашивать 2-3 раза подряд, прежде чем придёт реальный ответ
```

Титр (тот же текст симптома, для зрителя):

> «После перезагрузки VPS иногда первые один-два вопроса капитану Бладу
> получают шаблонный ответ вроде "капитан только проснулся, попробуй через
> полминутки" вместо настоящего — и это может повторяться, приходится
> спрашивать 2-3 раза подряд, прежде чем придёт реальный ответ»

Показать сам ввод промпта → работу агента (поиск причины по коду и
`deploy/captainblood.service`, правку `Main.kt`, сборку `./gradlew build`) →
финальный ответ в формате "Что нашёл / Что починил / Что проверил". Если
сборка реально долгая — ускорить этот фрагмент в монтаже с титром "ускорено
Nx", не вырезать/не подменять.

Титр рядом с результатом (по факту записи, прогон 1, 6м42с):

> Причина: `captainblood.service` зависел от `ollama.service` только по
> порядку запуска (`After=`/`Wants=`), не по готовности порта. При холодном
> старте VPS `Main.kt` проверял готовность Ollama один раз — при неудаче
> процесс завершался, а `systemd Restart=always` гонял полный передёрг JVM
> каждые 5с, пока Ollama не поднимется, раздувая окно шаблонного ответа.
> Фикс: `waitForOllamaServer()` (`Main.kt:1565-1624`) — до 12 попыток с
> паузой 5с (~55с) внутри ОДНОГО запуска JVM, без циклов рестарта процесса.
> `compileKotlin`+`build` зелёные, обычный режим (warm-up/health-check/
> обработка ошибок) не тронут.

## 4. Research — живой прогон (30-50 сек)

Промпт — вводится дословно в терминал (одной строкой):

```
@agent-research Как устроена авторизация запросов к капитану Бладу?
```

Титр: вопрос дословно — "Как устроена авторизация запросов к капитану
Бладу?"

Показать структурированный ответ (Файлы / Связи / Выводы). Титр (по факту
записи, прогон 2, 1м04с):

> HTTP Basic Auth (`BasicAuthenticator`), статичная пара логин/пароль из
> `config.properties` — токенов/сессий нет, креды заново на каждый запрос.
> `/chat` и `/` — единый realm (специально: браузер вводит креды раз,
> дальше сам подставляет в fetch к `/chat`). `/health` — сознательно без
> Auth, по чек-листу CLAUDE.md. Rate limit — только на `/chat`, срабатывает
> уже ПОСЛЕ Basic Auth (JDK-сервер проверяет Auth раньше вызова хендлера).
> Расхождений README/CLAUDE.md с кодом не нашёл.

## 5. Docs Sync — бонусный профиль, живой прогон (30-50 сек)

Промпт — вводится дословно в терминал (одной строкой):

```
@agent-docs-sync Синхронизируй README.md и CLAUDE.md с некоммиченными изменениями в рабочей копии проекта
```

Титр: контекст задачи — некоммиченный `Main.kt` (смена модели, countdown в
UI) + новый файл `docs/ModelComparison.md`, ещё не отражённые в `README.md`.

Показать итоговую правку `README.md` и `CLAUDE.md`. Титр (по факту записи,
прогон 3, 3м26с):

> README.md — абзац про новое поведение старта (ожидание Ollama ~55с) +
> раздел про выбор модели ответа со ссылкой на `docs/ModelComparison.md`.
> CLAUDE.md — обновлена диаграмма архитектуры под `waitForOllamaServer()`,
> структура папок, добавлены 2 новых паттерна (ретрай с паузой для внешней
> зависимости на старте; промпт-анкор анти-конфляции сущностей). Сам решил
> НЕ трогать `docs/AdvancedCourse_Day2_VideoScript.md` — по прецеденту
> такого же нигде не упомянутого файла Дня 1. Код не тронут.

## 6. Итог (15-25 сек)

Титр (по факту записи):

> Все три профиля сработали с первого прогона — доработка не потребовалась
> (в отличие от Дня 1, где v1 правил пришлось усиливать явным чек-листом).
> Прогон 1 (Bug Fix) — 6м42с, прогон 2 (Research) — 1м04с, прогон 3
> (Docs Sync) — 3м26с. Формулировка причины/фикса в Bug Fix разошлась с
> черновой проверкой в деталях (другая функция, другой механизм — не
> "прогрев без повтора", а "полный передёрг процесса systemd'ом"), но
> профиль справился сам, без вмешательства — черновая проверка была лишь
> ориентиром по тексту профиля, не готовым ответом.

Финальный титр:

> Узкий профиль с чек-листом "должен/не должен" и жёстким ограничением
> инструментов оказался надёжнее одного универсального ассистента — Research
> физически не может тронуть код (нет Edit/Bash), Bug Fix обязан
> скомпилировать перед ответом, Docs Sync не выдумывает то, чего нет в diff.

## Заметки на память

- Промпт для каждого живого сегмента (3-5) должен быть введён самим
  пользователем в его собственном терминале Claude Code, в реальном
  сеансе — не то же самое окно, где велось обсуждение сценария и черновая
  отладка профилей.
- Вызов профиля — зафиксировано `@agent-<имя> ...` (typeahead по имени) во
  всех трёх сегментах, дословные промпты — см. разделы 3-5 выше.
- **Важная техническая деталь**: черновая отладка (2026-07-21) проводилась
  через `general-purpose`-сабагентов с профилем, подставленным вручную в
  промпт (инструменты ограничивались текстовым указанием, не нативным
  frontmatter `tools:`) — так пришлось из-за того, что сессия-оркестратор
  была привязана к проекту `aiexperiment`, а не к `CaptainBlood`, и не видит
  `.claude/agents/*.md` этого репозитория как настоящие `subagent_type`.
  Для съёмки у пользователя открыт терминал именно в `CaptainBlood` — там
  профили должны подхватываться нативно (`@agent-bug-fix` и т.п.); если
  почему-то не подхватятся с первого раза — перед записью перезапустить
  Claude Code в этой директории (по спеке — новый `agents/`-каталог требует
  рестарта, дальше файлы обновляются на лету).
