## Проблема 1 — методологический артефакт тестовой оснастки (не баг сервиса)

**Где:** первая попытка сценария 2 (навигация через
`http://login:pass@localhost:8080/`).

**Симптом:** после входа в чате на любой вопрос выпадало «Не докричаться
до корабля — проверь соединение и попробуй снова.» (JS `.catch()` в
`Main.kt:870-872`), а баннер статуса показывал «нет связи с кораблём».

**Причина (подтверждена):** Chrome, управляемый Playwright через
`page.goto()` с credentials, встроенными в URL, не убирает их из
`location.href` (в отличие от навигации живым пользователем через нативный
диалог Basic Auth). Относительные `fetch("/chat")`/`fetch("/health")` в
JS страницы после этого падают по спецификации Fetch: `TypeError: Failed
to execute 'fetch' on 'Window': Request cannot be constructed from a URL
that includes credentials`. Прямой `curl -u ... /chat` при этом отработал
за 5с с корректным ответом — сам сервис не при чём.

**Обход в тесте:** пропатчен `window.fetch` через `browser_evaluate` —
резолвил относительные пути в абсолютные + добавлял `Authorization`
заголовок явно (эмуляция того, как реальный браузер кеширует Basic Auth
после диалога). После этого сценарии 2/4/5 отработали штатно.

**Не чинить в коде сервиса** — это артефакт способа навигации в
автоматизированном тесте, реальный ребёнок логинится через нативный диалог
браузера, где `location.href` credentials не содержит.

## Проблема 2 — языковой дрейф модели на офтопик-вопросе (реальный, но известный баг)

**Где:** Сценарий 5, первая попытка, вопрос «Как приготовить пиццу?».

**Симптом:** вместо ожидаемого отказа/переключения на морскую тему в чат
пришла техническая заглушка: «Не знаю. Ответ модели содержал недопустимые
символы (языковой дрейф на другой алфавит), не удалось получить
корректный ответ на русском. Попробуйте переформулировать вопрос.»
(`OllamaClient.kt:293`, срабатывает после исчерпания
`MAX_LANGUAGE_DRIFT_RETRIES` попыток).

**Гипотеза причины:** известная особенность семейства моделей `qwen2.5`
(зафиксирована в `~/.claude/rules/SETUP_ENVIRONMENT.md`) — на коротких/
неоднозначных промптах модель иногда уезжает в CJK-алфавит вместо
русского. `RUSSIAN_LANGUAGE_GUARD` + ретраи (`OllamaClient.kt:314-325`)
обычно это лечат, но не гарантированно — здесь все попытки не помогли.

**Не проблема keyword-gate/офтопик-логики как таковой** — повторный запрос
с другим офтопик-вопросом («Расскажи про футбол») отработал штатно и дал
адекватный увод темы в морской контекст. Это флаки-поведение конкретной
генерации модели, не системная логическая ошибка. Не правил код — фикс
вне явного запроса, сообщаю по правилу проекта.
